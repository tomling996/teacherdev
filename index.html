提示詞範例 1：建構核心架構

設計原意：確立平台的基本邏輯，參考現有技術機制。

我希望建立一個專為中學編程教學設計的課堂評測網站，核心機制請參考 LeetCode 等「自動化評測平台 (Online Judge)」。系統需具備以下核心架構：

1. 題目建立：教師可預先設定題目，輸入標準 Python 程式碼後，系統需能自動生成或驗證「測試用例 (Test Cases)」。
2. 自動評改：學生提交課堂練習後，系統需即時在瀏覽器端執行其程式碼，比對測試結果並自動核發分數。
3. 數據追蹤：教師後台需能實時檢視學生的測試紀錄、提交的程式碼及最終得分，以作持續性評估。

如需進一步補充系統規格或技術細節以完善平台開發，請向我提問。

提示詞範例 2：功能優化與除錯

設計原意：解決課堂實際應用時出現的系統異常與效能問題。

請在維持現有架構不變的前提下，為系統進行以下優化與除錯：

【功能優化】
1. 教師後台：於題目管理介面新增「依日期篩選」功能。
2. 教師後台：將學生提交紀錄優化為「實時動態更新 (Real-time update)」，移除手動重新整理按鈕。
3. 效能防護：針對學生可能誤寫的無窮迴圈 (Infinite Loop) 加入防護機制，將單次迴圈執行間隔設為 0.1 秒，避免導致網頁崩潰。
4. 執行控制：新增「強制終止」按鈕。當執行程式時，按鈕動態切換，運行結束後自動恢復為「測試程式碼」按鈕。

【異常修復 (Bug Fixes)】
5. 視窗異常：修復執行 `while True: a = input()` 時，導致系統無限觸發輸入彈跳視窗的問題。
6. 型別比較防護：修復因字串與整數進行大小比較所引發的 `TypeError: '<' not supported...` 系統報錯。

請輸出更新後的完整 HTML 程式碼。

提示詞範例 3：資料驗證與後台擴充

設計原意：完善評分邏輯，並加強學生輸入資料的格式控制。

請在現有的基礎上，協助完善系統評分邏輯及資料驗證 (Data Validation) 機制：

【後台擴充與邏輯修復】
1. 計分異常：修正答案全對但系統仍核發 0 分的邏輯錯誤。
2. 資料庫管理：於教師專區擴充 Firebase 數據管理介面 (Data Management UI)，以便直接檢視與維護所有資料表。
3. 篩選機制：擴充現有題目與日期的篩選器，加入「顯示所有日期 (All Dates)」的選項。

【前端輸入驗證 (Input Validation)】
4. 班別驗證：限制輸入格式為兩位大寫字符（僅限 1A-1E 至 6A-6E），系統需主動阻擋非規範輸入。
5. 學號驗證：限制輸入格式為兩位數字字串（範圍 01 至 45），系統需主動阻擋非規範輸入。

請輸出更新後的完整 HTML 程式碼。
